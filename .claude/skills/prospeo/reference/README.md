# Prospeo enum reference

Snapshot of every enum the Prospeo Search API validates against. Use these files as the source of truth when building `filters.json` payloads — Prospeo rejects unknown values with `INVALID_FILTERS`, and a wrong guess costs a roundtrip. Always grep here first.

## Files

| File | Filter key | Lines | Notes |
| --- | --- | --- | --- |
| [seniorities.txt](seniorities.txt) | `person_seniority.include` | 10 | `Founder/Owner` is one literal value with the slash, not two. |
| [departments.txt](departments.txt) | `person_department.include` AND `company_headcount_growth.departments` | 195 | Two sub-enums separated by section headers (`# Normal Departments` and `# Headcount Growth Departments`). They are NOT interchangeable — pick the section that matches the filter key. Misspellings like `IT Recruitement` and `Opthalmology` are preserved verbatim because the API expects them as-is. |
| [industries.txt](industries.txt) | `company_industry.include` / `.exclude` | 259 | LinkedIn taxonomy. Use exact strings like `Software Development`, `IT Services and IT Consulting`, `Financial Services`. **Not** legacy aliases like `Software` or `Computer Software`. |
| [employee-ranges.txt](employee-ranges.txt) | `company_headcount_range` | 11 | Bands only. `11-20`, `21-50`, ..., `5001-10000`, `10000+`. Arbitrary ranges like `11-50` are rejected — combine adjacent bands instead. Note: top three bands are `1001-2000`, `2001-5000`, `10000+` (NOT `1001-5000`/`5001-10000`/`10001+`). |
| [funding-stages.txt](funding-stages.txt) | `company_funding.stage` | 23 | `Pre seed`, `Seed`, `Series A` … `Series E-J`, plus event types like `Angel`, `Grant`, `Debt financing`, `Post IPO equity`, `Convertible note`, etc. Capitalization and spacing matter (`Pre seed` not `Pre-Seed`). |
| [mx-providers.txt](mx-providers.txt) | `company_email_provider` | 102 | Examples: `Google`, `Microsoft`, `ProtonMail`, `IONOS`, `OVH`, `Zoho Mail`, `Yandex Mail`. |
| [technologies.txt](technologies.txt) | `company_technology.include` | 4,940 | Big list. Grep before writing the filter, e.g. `grep -i snowflake technologies.txt`. |
| [naics-codes.txt](naics-codes.txt) | `company_naics.include` | 1,012 | 6-digit codes, tab-separated `CODE\tDESCRIPTION`. Pulled from the 2022 NAICS census source Prospeo's docs link to. Pass codes as integers in the filter (e.g. `541511`). |
| [sic-codes.txt](sic-codes.txt) | `company_sics.include` | 1,005 | 4-digit codes, tab-separated `CODE\tDESCRIPTION`. Pass codes as integers/strings in the filter. |

## Filters NOT covered (free-text, validated by separate API)

- `person_location_search` / `company_location` — strings only, must match Prospeo's location dictionary exactly. Use the [Search Suggestions API](https://prospeo.io/api-docs/search-suggestions) to validate, or copy values from the dashboard payload.
- `person_job_title` — strings only, no enum. Use Search Suggestions API to validate.
- `person_year_of_experience` — `{min, max}` integers (0–60). No enum.

## How to use these files

Before writing `filters.json`, look up each enum value here. A safe workflow:

```bash
REF=".claude/skills/prospeo/reference"

# Industry: confirm "Software Development" is a valid value
grep -i "^software development$" "$REF/industries.txt"

# Headcount: see all valid bands
cat "$REF/employee-ranges.txt"

# Technology: find the canonical capitalization for "snowflake"
grep -i "snowflake" "$REF/technologies.txt"

# NAICS: find code for "Custom Computer Programming"
grep -i "custom computer programming" "$REF/naics-codes.txt"
```

If the API ever rejects a filter with `INVALID_FILTERS`, re-grep here before guessing alternatives. If a value is missing from these files but Prospeo accepts it, treat that as a sign the snapshot is stale and refresh.

## How to refresh

These files were generated by parallel WebFetch agents against `https://prospeo.io/api-docs/enum/<name>` (and `census.gov` for NAICS). Re-running the same agent prompts will regenerate them. The technologies page is 14MB+ — use `curl + gunzip` and parse the embedded Next.js data stream instead of WebFetch (see `technologies.txt` first-fetch notes if anything looks off).
